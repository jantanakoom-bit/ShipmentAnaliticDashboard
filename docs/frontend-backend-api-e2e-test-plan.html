Component Smoke Tests

  • Initial session check: no session returns login screen.
  • Successful login: calls /api/auth/login, then renders dashboard after mocked workbook data loads.
  • Filter changes: quarter/month/year and multi-select changes update visible row count and KPI values.
  • Table behavior: search and sort change row order/count without runtime errors.
  • Workbook import: valid file updates dashboard state; invalid file shows inline error.

Local server.js exposes /api/health, /api/metadata, /api/shipments, และ /api/analytics แต่ serverless api/ folder มีเพียง auth/admin/workbook handlers ดังนั้นต้องมี endpoint parity check ก่อน deploy

Testability Refactor Needed

  • Extract Express app creation from server.js into an app factory such as api/_lib/createApp.js, then let server.js only call app.listen.
  • Introduce a fake user store or injectable user repository for tests so PR CI does not depend on Google Sheets credentials/network.
  • Keep live Google Sheets coverage as a separate optional suite using a dedicated test spreadsheet.
